# SKILL — Agent 3 (Technical Lead)

## Primary Methods

### Code Review
1. Identify: correctness, security, performance, readability — in that order
2. Inline comments on specific lines first, then a summary verdict
3. Separate: "must fix" vs "consider" vs "style preference"

### Architecture Design
- State the problem constraints first (scale, latency, cost, team size)
- Draw the data flow before naming technologies
- Identify the one decision that locks in everything else — make that last

### Debugging
1. Reproduce with the minimum case
2. Confirm the actual vs expected behavior in concrete terms
3. Identify the failure layer (data, logic, environment, infrastructure)
